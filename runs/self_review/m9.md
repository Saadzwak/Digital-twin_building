## Posture du juge hostile

**Quelle affirmation ne survivrait pas à un examen ?** Une interface avec des
emplacements de graphique n’est pas un dashboard de résultats tant que M4
échoue. Correction : l’écran réel s’arrête après le message de blocage; il ne
montre aucune identité, perte, dérive, contrefactuel ou classement inventé.

**Quel chiffre est affiché sans incertitude ?** Le contrat interdit une
estimation sans valeur, intervalle, unité, période, méthode et source. Les
seuls textes actuels du dashboard sont le statut de blocage et le chemin de
diagnostic, pas des résultats chiffrés.

**Quelle hypothèse est enterrée dans le code ?** Un intervalle bootstrap ne
transforme pas le BIC en probabilité, ni une branche effective en mur. Correction :
le modèle de données nomme des « effective thermal paths » et rejette les
méthodes contenant wall/façade/paroi; les contrefactuels sont explicitement
conditionnels.

**Qu’est-ce qui marche uniquement sur ce jeu de données précis ?** La lecture
de `runs/m4/verdict.json` et la structure future de payload. Correction : les
contrats demandent une source de run par chiffre plutôt que de déduire une
valeur d’un nom de fichier.

**Où ai-je écrit « ça marche » sans l’avoir exécuté ?** Nulle part : compilation,
import, démarrage/arrêt contrôlé et tests sont tous exécutés. Je ne revendique
pas le rendu visuel d’une charge utile valide, puisqu’il n’en existe pas.

correct est de bloquer le produit plutôt que d’afficher une précision fictive.
