## Posture du juge hostile

**Quelle affirmation ne survivrait pas à un examen ?** « Le noyau reproduit
4R3C » est vrai seulement lorsque les paramètres du notebook sont injectés ;
ce n’est pas une reproduction de l’identification. Correction : le verdict
M4 est explicitement `validated=false`, et aucune couche produit ne peut le
transformer en succès.

**Quel chiffre est affiché sans incertitude ?** Les RMSE/BIC du diagnostic
sont des sorties déterministes de non-régression, pas des estimations produit.
Correction : le dashboard/chat ne les affichera pas comme résultats utilisateur
sans intervalle empirique, et refuse totalement un run M4 non validé.

**Quelle hypothèse est enterrée dans le code ?** Le runtime de l’exécution
notebook n’est pas connu; pandas/SciPy/BLAS peuvent changer la trajectoire de
L-BFGS-B. L’agrégation Tin streaming conserve la formule mais pas
nécessairement l’ordre de somme bit-à-bit. Correction : les deux hypothèses,
leur priorité et les tests d’exclusion sont écrits dans le diagnostic M4.

**Qu’est-ce qui marche uniquement sur ce jeu de données précis ?** Le test des
paramètres notebook est une forte preuve de cohérence pour cette archive, pas
une garantie de portabilité à un nouveau bâtiment ou une nouvelle version de
PLEIAData. Correction : il est un oracle de régression, jamais une promesse
physique.

**Où ai-je écrit « ça marche » sans l’avoir exécuté ?** Nulle part. La seule
conclusion positive est limitée aux tests qui ont été exécutés. La conclusion
de reproduction est négative et les 19 valeurs observées sont sauvegardées.
